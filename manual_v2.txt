# QR Attendance System Installation Guide (manual_v2)

This guide explains how to install and run the QR Attendance system on a new device, **without copying any database data**. Follow these steps to set up a fresh instance.

---

## 1. Prerequisites
- **Windows 10/11** (recommended)
- **Python 3.11+** (download from python.org)
- **Git** (optional, for cloning the repo)

## 2. Obtain the Source Code
- Copy the project folder to the target device **excluding any database files**:
  - Do **NOT** copy `attendance.db` or any backups (e.g., `instance/attendance.db.bak`).
- Alternatively, clone from your repository:
  ```powershell
  git clone <your-repo-url>
  ```

## 3. Set Up Python Environment
- Open PowerShell in the project folder.
- Create and activate a virtual environment:
  ```powershell
  python -m venv .\venv
  .\venv\Scripts\Activate.ps1
  ```

## 4. Install Dependencies
- Install required packages:
  ```powershell
  pip install flask flask-bcrypt flask-login flask-sqlalchemy psycopg2-binary python-dotenv "qrcode[pil]"
  ```
  Or, if you have `requirements.txt`:
  ```powershell
  pip install -r requirements.txt
  ```

## 5. Initial Configuration
- **No database file is present:**
  - On first run, the app will create a new, empty `attendance.db` in the project root.
- (Optional) Set environment variables for production:
  ```powershell
  $env:FLASK_SECRET_KEY = 'your-secret-key'
  $env:DATABASE_URL = 'sqlite:///attendance.db'  # Or your preferred DB
  ```

## 6. Run the Application
- Start the server:
  ```powershell
  python .\app.py
  ```
- The app runs on port 5000 by default. Access via browser at `http://localhost:5000`.

## 7. Create Initial Accounts
- Use the web interface to create student and teacher accounts.
- To bootstrap admin access, log in as:
  - **Email:** `admin@teacher`
  - **Password:** `system123`
  - This will create the admin teacher account if it does not exist.

## 8. Notes
- **Do not copy any old `attendance.db` files** if you want a fresh start.
- All static HTML pages are served directly; styles are in `CSS/`.
- For desktop app setup, see `DESKTOP_APP_SETUP.md`.
- For troubleshooting, see `BUILD_GUIDE.md` and `DEPLOYMENT.md`.
